⚙️ Setup Guide — Connect Firebase (Free)

Step 1 — Create a Free Firebase Project
  1. Open console.firebase.google.com in your browser
  2. Sign in with your Google account
  3. Click "Add project" → name it Attendance System
  4. Disable Google Analytics → click "Create project"
  5. Wait ~30 seconds for it to be ready, then click "Continue"
Step 2 — Enable Firestore Database
  1. In the left sidebar, click "Firestore Database"
  2. Click "Create database"
  3. Choose "Start in production mode" (we'll add proper rules in Step 7)
  4. Select your nearest region (e.g. asia-southeast1 for Singapore) → click "Enable"
Step 3 — Enable Firebase Authentication (Required for Admin Login)
  1. In the Firebase Console left sidebar, click "Authentication"
  2. Click "Get started"
  3. Under "Sign-in method", enable "Email/Password"
  4. Go to the "Users" tab → click "Add user"
  5. Enter your admin email and a strong password (min 12 chars) → click "Add user"
  6. This email/password is what you use to log into the dashboard. No credentials are stored in the HTML file.
Step 4 — Get Your Config Keys
  1. Click the ⚙️ gear icon (top-left) → "Project settings"
  2. Scroll down to "Your apps" section
  3. Click the </> Web icon → type any app name → click "Register app"
  4. You'll see a firebaseConfig block — copy the whole object
Step 5 — Paste Config Into This File

Open attendance_system.html in Notepad (or any text editor). Find var FIREBASE_CONFIG = { near the bottom of the file and replace the placeholder values with your actual Firebase config.

var FIREBASE_CONFIG = {
  apiKey: "AIzaSy...",
  authDomain: "your-project.firebaseapp.com",
  projectId: "your-project-id",
  storageBucket: "your-project.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abc123"
};

⚠️ Never share your HTML file publicly with the real API key visible — restrict it in Google Cloud Console to your domain only (see security guide).

Step 6 — Update Your Work Sites

In the same file, find const WORK_SITES = [...] and edit each entry with your actual site names and GPS coordinates.

To get GPS coordinates: open Google Maps → right-click on your site location → the first line shown is latitude, longitude.

Step 7 — Set Firestore Security Rules (🔒 Critical)

In Firebase Console → Firestore Database → Rules tab, replace all existing rules with:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /attendance/{docId} {
      // Engineers can check in (create record) without login
      allow create: if request.resource.data.keys().hasAll([
                      'empId','empName','site','checkinTimestamp','status'])
                    && request.resource.data.empId is string
                    && request.resource.data.empId.size() > 0
                    && request.resource.data.empId.size() <= 20
                    && request.resource.data.empName is string
                    && request.resource.data.empName.size() > 0
                    && request.resource.data.empName.size() <= 60
                    && request.resource.data.status == 'in';

      // Engineers can check out (update only allowed fields)
      allow update: if request.resource.data.diff(resource.data)
                      .affectedKeys().hasOnly([
                        'checkoutTime','checkoutISO','checkoutTimestamp',
                        'checkoutLat','checkoutLng','checkoutDist',
                        'checkoutAcc','status','duration','isOvertime'])
                    && request.resource.data.status == 'out';

      // Only signed-in admin can read or delete records
      allow read, delete: if request.auth != null;
    }
  }
}

✅ This ensures only your admin can view data. Engineers can only write their own records.

Step 8 — Restrict Your Firebase API Key to Your Domain
  1. Go to console.cloud.google.com
  2. Select your Firebase project from the top dropdown
  3. Navigate to APIs & Services → Credentials
  4. Click on your Browser key (auto created by Firebase)
  5. Under "Application restrictions", select "HTTP referrers"
  6. Add your domain: https://YOUR_USERNAME.github.io/*
  7. Click Save

This prevents anyone from using your API key on other websites.

Step 9 — Host for Free (Get a Shareable URL)
  1. Go to app.netlify.com/drop
  2. Drag and drop your attendance_system.html file onto the page
  3. Wait 10 seconds — you'll get a free URL like https://amazing-name-123.netlify.app
  4. Share that URL with all your engineers!

✅ That's it! No coding skills needed. Completely free.

Firebase Free Tier limits (Spark plan — always free):
1 GB storage · 50,000 reads/day · 20,000 writes/day — more than enough for 100 engineers.
Step 10 — Add Leave Application Firestore Rules

In Firebase Console → Firestore Database → Rules tab, add the leaveApplications collection rules inside the existing match /databases/{database}/documents block:

match /leaveApplications/{docId} {
  // Any engineer can submit a leave application
  allow create: if request.resource.data.keys().hasAll([
                  'empId','empName','empEmail','leaveType',
                  'fromDate','toDate','numDays','status'])
                && request.resource.data.empId is string
                && request.resource.data.empId.size() > 0
                && request.resource.data.status == 'pending';

  // Engineers can read leave records (to check status)
  allow read: if true;

  // Only signed-in admin can update (approve/reject) or delete
  allow update, delete: if request.auth != null;
}

✅ This allows engineers to submit and view leave applications. Only your admin can approve or reject them.

Step 11 — Configure Email Notifications (Optional)

To enable automatic email notifications when leaves are approved/rejected:

  1. Sign up for free at emailjs.com
  2. Create an Email Service (connect your Gmail)
  3. Create an Email Template with variables: {{to_email}}, {{to_name}}, {{message}}, {{leave_type}}, {{status}}
  4. Get your Public Key, Service ID, and Template ID
  5. In this file, find var EMAILJS_KEY near the top of the script and replace the placeholder values

If EmailJS is not configured, the system will show a "Send via Email Client" fallback button that opens your email app with the notification pre-filled.
